# 音声アセット生成メモ（ElevenLabs ＋ 分割スクリプト）

`phrase_01.mp3`〜`phrase_48.mp3`（2026-06-28 配置）を作成した際の手順とパラメータの記録。
再生成・追加フレーズ作成時の再現性のために残す。命名規則・id対応表は [README.md](README.md) を参照。

## 1. 概要
ElevenLabsでの音声合成と、Claude Codeによる無音検知スクリプトでの分割を組み合わせて作成。
48フレーズを1本の音声として一括生成し、その後1文ずつに自動分割した。

## 2. ElevenLabs 生成パラメータ（完全再現用）
再生成や新規フレーズ追加時は、声質・トーンのばらつきを防ぐため以下を完全に一致させる。

| 項目 | 値 |
|---|---|
| Voice（話者） | `Ninoska - Spanish Teacher` |
| Model（AIモデル） | `Eleven Multilingual v2` |
| Speed（話速） | `0.8`（初学者向けにやや遅め） |
| Stability（安定性） | `65%` |
| Similarity boost（明瞭性・類似性） | `75%` |
| Style（スタイル過大評価） | `0%` |
| Speaker boost（話者強調） | Enabled |

## 3. 作成手順（バルク生成・分割方式）
1. 48フレーズ（仕様書の正解文）を改行区切りで1テキストブロックにまとめ、ElevenLabsへ一括投入。
   - **疑問文だけ末尾に `???` を付与**（下記4章のTTSハック。生成用テキストのみの処置で、アプリ内の文章データ自体は通常の `¿ ... ?` のまま）。
2. 上記パラメータで、48フレーズが連続する1本の音声（約1分35秒）を生成・ダウンロード。
3. 市販のWebトリミングツールでは単語間の細かな息継ぎで誤分割が発生したため、Claude Code環境で**無音（Silence）区間をミリ秒単位で判定するスクリプト**を実行し、1文も欠けることなく48個の個別mp3に一括分割。
4. 分割後のファイルを `phrase_01.mp3`〜`phrase_48.mp3` にナンバリングし、`/assets/audio/` に配置（id 1〜48と1対1対応、対応表は [README.md](README.md)）。

## 4. 疑問文のTTSハック（`???`）
ElevenLabsの多言語モデルは、通常の疑問符（`¿ ?`）だけでは語尾の上がり（疑問調イントネーション）が弱くなることがあった。
そのため**生成時の投入テキストに限り**、疑問文の末尾を `???`（クエスチョンマーク3連）にすることで、AIに強い疑問調と認識させ、ネイティブらしい質問のイントネーションに補正した。

実際に適用した6フレーズ（id 2, 11, 18, 27, 35, 47）：
- `¿Tú hablas inglés???`
- `¿Ustedes viven en Barcelona???`
- `¿Tú bebes vino???`
- `¿Ustedes compran un regalo???`
- `¿Vosotros escribís una postal???`
- `¿Usted compra un regalo???`

> 注：これは生成時のテキスト入力のみのハック。`index.html` 内のマスターデータやアプリ画面表示は通常の `¿ ... ?` 表記のまま変更していない。

## 5. 今後フレーズを追加・再生成する場合の手順
1. 追加したいフレーズを改行区切りで並べる。疑問文は末尾に `???` を付与（生成用テキストのみ）。
2. 上記2章のパラメータ（Ninoska, Multilingual v2, Speed 0.8, Stability 65%, Similarity 75%, Style 0%, Speaker boost ON）を完全にコピーして生成。
3. Claude Codeで無音検知ロジック（ミリ秒・dB指定）による分割を再現し、個別アセットを切り出す。
4. `phrase_{XX}.mp3` の連番命名を維持し、`assets/audio/README.md` の対応表も更新する。再生は回答後（`isAnswered === true`）に有効化される仕様（[index.html](../../index.html)）と同期させる。
